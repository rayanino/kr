Reviewed DR28 IU-6 (classify) and IU-7 (enrich) prompt architecture on 2026-04-07.

**Key findings:**
- All 3 call sites (classify, group, enrich) correctly use system=CONSTITUTION. Pattern is consistent.
- Cache keys correctly use CONSTITUTION as system component.
- SPEC §7.2.2 code block has significant drift from actual _ENRICH_RULES code: narrator role missing, CONSERVATISM RULE missing (threshold 0.3 vs SPEC's 0.5), 6 additional epithets missing, 2 worked examples missing.
- ENRICH_CRITICAL_REMINDERS content is not documented in any SPEC code block (§5.2.3 documents CLASSIFY reminders but no equivalent for ENRICH).
- test_phase3_enrichment.py lacks system message content verification (unlike test_phase2_classify.py which checks CONSTITUTION-only and no task rule leakage).

**Why:** SPEC code block drift is the #1 risk pattern in this project (confirmed across 3 reviews now). The confidence threshold mismatch (code 0.3 vs SPEC 0.5) could cause behavioral divergence if someone reimplements from SPEC.

**How to apply:** When reviewing any prompt-related changes, always diff the SPEC code block against the actual code constant. Check for additions in code not reflected in SPEC. Confidence thresholds are especially dangerous to have mismatched.
