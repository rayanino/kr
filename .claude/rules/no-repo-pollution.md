- **NEVER commit temporary artifacts to the repository.** This includes handoff files, DR prompts, DR response archives, analysis reports, session notes, characterization reports, sampling reports, or any document whose findings have been (or will be) incorporated into spec atoms or code. The spec atoms and code ARE the authoritative record. Temporary documents are redundant pollution.
- **Specifically banned file patterns:** `*HANDOFF*`, `*handoff*`, `DR_*.md`, `*dr-prompt*`, `*characterization*`, `*sampling*`, `*session_report*`, `*analysis_report*`. If a file matches one of these patterns, it does NOT belong in the repo.
- **Where temporary context goes:** Session handoff context → memory system (`~/.claude/projects/*/memory/`). DR findings → spec atoms or code amendments. Analysis results → spec atom amendments. DR prompts → ephemeral (owner pastes them, they're consumed).
- **The test:** Before committing any `.md` file that is not `README.md`, `CLAUDE.md`, `AGENTS.md`, or `SPEC_OVERVIEW.md`, ask: "Will this file still be useful in 6 months, or is it a snapshot of today's work?" If it's a snapshot, it does NOT belong in the repo.
- **Why this exists:** V1 polluted the repo to 16,450 files. Session 23 cleaned it to 6,339. The source engine spec session immediately re-polluted with 12 temporary files. The owner caught it. This is a zero-tolerance rule. Owner directive 2026-04-15.
