## 1. CURRENT STATE: What Works vs. What's Broken

**The deterministic infrastructure is solid.** 148 tests pass, zero failures. The engine handles tree loading (both v0 and v1 YAML formats), excerpt type classification, confidence-based routing, tie detection, provenance preservation (D-023), Arabic text byte-identical verification (T-1), input validation, batch diagnostics with four warning conditions, and file output with correct paths and serialization. Every module (`tree_loader.py`, `placer.py`, `validator.py`, `writer.py`, `diagnostics.py`, `engine.py`) is implemented and tested. All 5 trees load correctly: nahw (183), sarf (226), balagha (335), imlaa (105), aqidah (30).

**But the engine has no brain.** Session 2 was never built. `engines/taxonomy/src/llm_adapter.py` does not exist. The `PlacementAdapter` protocol is defined in `placer.py` with a `StubPlacementAdapter` that raises `NotImplementedError` for all LLM calls. The engine catches this in `_process_excerpt()` (line ~240 of `engine.py`) and routes everything to "unplaced" with reason `"LLM placement not available (Session 2)"`. The taxonomy engine, in its current state, is a skeleton that can load trees and write files but cannot classify a single excerpt.

**Four additional broken items:**

- **Gold baseline completely invalid** (`engines/taxonomy/tests/fixtures/gold_baseline_nahw.yaml`): All 13 entries reference v1.0 leaf paths (e.g., `almajrurat/huruf_aljar/huruf_aljar__overview`, `almajrurat/huruf_aljar/ma3ani_huruf_aljar`). None of these paths exist in the nahw v2.0 tree. The v2.0 paths use different transliteration (`al_jarr_wa_al_idafa/huruf_al_jarr/maani_huruf_al_jarr_al_asliyya`). Verified: 13/13 MISSING.

- **SPEC section 9.1 stale**: States "Leaf counts: 226, 226, 335, 30, 105" — the nahw count should be 183 (v2.0 was installed at commit `8c7898cf` on 2026-04-01, but the SPEC was never updated).

- **`tracer.py` is dead code**: 113 lines that hardcode all excerpts to `nahw/kalaam/definition`. Predates the real engine. Should be deleted.

- **4 of 5 installed trees are unvalidated originals**: The registry points sarf to `sarf_v1_0` (226 leaves), balagha to `balagha_v1_0` (335 leaves), aqidah to `aqidah_v0_2` (30 leaves), imlaa to `imlaa_v1_0` (105 leaves). Per `TAXONOMY_TREE_PROTOCOL.md` section 0 (Failure history): "All v1_0 trees were generated by AI without documented methodology, without independent verification, without corpus validation, and without owner review."
