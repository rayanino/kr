# DR Prompt — Display Metadata Design for Teaching Units

**Target:** Gemini DR (needs file uploads — prepare a bundle)
**Purpose:** Design the "source card" that appears alongside teaching units — what information about an author and book should the reader see?

---

## Prompt for the owner to paste into Gemini DR:

I am building a personal Islamic scholarly library (خزانة ريان) that processes Arabic scholarly books into "teaching units" — self-contained excerpts designed for study. When a reader sees a teaching unit, they should also see a "source card" that tells them about the author and the book.

**Context:** The library contains classical Islamic scholarly works across hadith, fiqh, tafsir, aqidah, nahw, sarf, and more. The collection includes everything from foundational texts (Sahih Bukhari, Tafsir Ibn Kathir) to small risalahs and modern compilations. The library's core principle is ZERO KNOWLEDGE LOSS — nothing is ever hidden, compressed, or simplified.

**Your task:** Design the ideal "source card" that appears alongside teaching units. Consider:

### 1. Author information
- What should the reader see about the author?
- Is the full name (ism + nasab + kunya + laqab + nisba) always shown, or is there a readable display form with the full name available on expansion?
- Should the death date be shown in Hijri only, or both Hijri and Gregorian?
- Should the author's scholarly standing (الحافظ, الإمام, شيخ الإسلام) be displayed?
- Should the author's madhab affiliation be shown?
- Should there be a short biographical blurb? If so, what should it contain for different types of scholars?
  - Classical hadith scholars (e.g., البخاري)
  - Classical fiqh scholars (e.g., ابن قدامة)
  - Polymaths (e.g., السيوطي)
  - Modern scholars (e.g., ابن باز)
  - Unknown/anonymous authors

### 2. Source/book information
- What should the reader see about the book itself?
- Should the genre classification be displayed? In what form?
- Should the science(s) be displayed?
- For multi-layer works (sharh on matn), should the reader see the layer structure?
- Should edition/publisher information appear?
- Should there be a blurb about the book's significance?
- For hadith collections: should the specific hadith subgenre (musnad, sunan, jami, etc.) be shown?

### 3. Trust and provenance
- Should the reader see trust information? In what form?
- If authorship is disputed, how should the dispute be displayed alongside the excerpt?
- Should the reader see where the metadata came from (which databases verified it)?

### 4. Islamic scholarly tradition alignment
- In the Islamic scholarly tradition (حلقات العلم, إجازات), how do scholars traditionally introduce a book before teaching from it? What information do they give their students about the author and text?
- How should the source card mirror this traditional scholarly introduction?
- Are there elements from the Islamic scholarly tradition that digital libraries typically miss?

### 5. Design principles
- The reader is a Muslim student with minimum Islamic knowledge. The card must be educational, not assume prior knowledge.
- Everything must be in Arabic (the library is Arabic-first).
- The card should serve both quick-glance reading (short form) and deep-dive research (expanded form).
- When there is scholarly disagreement about any field, ALL positions must be visible (zero knowledge loss).

Please provide: (a) a recommended source card structure with field names and descriptions, (b) 3 concrete examples for different book types (classical hadith, multi-layer sharh, modern risalah), and (c) what information should be generated by agent teams vs extracted from existing databases.
