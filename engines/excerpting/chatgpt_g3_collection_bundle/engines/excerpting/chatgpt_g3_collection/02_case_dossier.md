# Purpose and scope

This bundle evaluates a numbered-point case from a questionnaire. It is not promoted doctrine. It preserves:

- an owner-faithful answer to the questionnaire
- a separate engineering/protocol expansion

# Explicit separation of layers

## Owner-faithful questionnaire layer
The owner’s answer is not “numbered points are always separate” and not “numbered lists should stay together.” The strongest owner-faithful answer is content-first: numbering itself should never decide excerpting, but individual numbered points often do end up separate because their content warrants it.

## Expanded engineering / protocol layer
The deeper protocol issue is that numbering is a dangerous false criterion, while function, topic, and placement pressure are the real criteria. Numbered-list extraction also creates extra reassurance burdens that the library must handle better than books.

# Core assessment thesis

The strongest answer is **C**.

That is not because the owner is undecided. It is because the rule is not “separate by numbering” or “keep by numbering.” The rule is: analyze the content, the functions, and the benefit of the resulting excerpt. Point 2 likely deserves its own excerpt, but because of its content and explanation, not because it is point 2.

# Why numbering is a dangerous false criterion

Numbering is authored structure, not excerpt identity. If numbering becomes decisive, the system will:
- split things that should stay together
- keep together things that should split
- confuse presentation structure with knowledge structure

That is why the owner reacted so strongly against the form of the question itself.

# Why content/function-first excerpting is the real rule

The real deciding questions are:
- what topic does this passage address?
- what function(s) does it perform?
- does it gain value as a separate unit?
- does it need attached explanation to remain useful?
- does it legitimately belong under multiple leaves?

# Why numbered points often still end up separate

Numbered points often end up separate because:
- each point often states a distinct ruling or inference
- each point may carry its own explanation
- each point may map to a specific leaf better than the list as a whole

So “often separate” is a descriptive outcome, not a governing rule.

# Multi-function point doctrine pressure

Point 1 is a useful example from the surrounding context. It is not merely “one list item”; it also:
- explains the hadith
- repeats or crystallizes a ruling

That means a point may need more than one placement. The same pressure can apply to point 2 if it simultaneously belongs to:
- hadith-explanation analysis
- ruling/disagreement structure

# Context / reassurance pressure from numbered lists

Numbered extraction raises immediate questions:
- why this point and not the previous one?
- how does point 2 relate to point 1 and point 3?
- does the list introduction matter?
- is this point being read without the author’s intended frame?

So numbering is not a split rule, but it is a reassurance burden.

# Library-vs-books methodology pressure

The owner’s raw comments reveal a deeper adoption risk: if the library does not provide enough benefit and reassurance over books, the owner may simply prefer reading books. This case matters because crude numbered-point extraction could make the library feel like chopped book fragments rather than a superior study environment.

# Candidate packaging directions

1. **Point as own excerpt**
   - works when the point states a distinct issue and its continuation belongs tightly with it

2. **Point plus attached explanation**
   - likely the right local packaging for point 2

3. **Merge all points**
   - too coarse when points are genuinely different rulings

4. **Duplicate across relevant leaves**
   - valid when a point carries multiple functions and belongs under more than one leaf

5. **Retain parent-list context**
   - useful for reassurance, but should not override content-first excerpting

# What would be reckless to flatten

- flattening “numbered” into an excerpting rule
- flattening “usually separate” into “always separate”
- flattening content/function analysis into list-position analysis
- flattening multi-function points into single-placement objects
- flattening reassurance needs into optional polish

# What would be reckless to automate blindly

- “one numbered point = one excerpt” as a hard rule
- “all points from one list stay together” as a hard rule
- ignoring the list introduction and neighboring points when reassurance depends on them
- failing to detect multi-function placement
- building a library whose handling of such cases offers too little benefit over books
