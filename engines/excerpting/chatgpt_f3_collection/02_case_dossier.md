# Core assessment thesis

This passage fails as one standalone entry because it is **functionally overloaded**. It is introduced by chapter-start language that makes it look like a pure introduction, but the text actually performs several jobs at once:
- chapter introduction
- ruling statement
- proof overview
- refutation

That means the main local problem is not “missing information” in the simple sense. The deeper problem is **mispackaging**: one passage is carrying multiple excerpt-worthy functions.

# Surface impression vs actual function

## Surface impression
The sequence:
- chapter title
- “هذا الباب يذكر فيه شيء من أدلة مشروعية المسح على الخفين”

strongly invites a reader to treat the passage as a chapter introduction.

## Actual function on inspection
When the source continuation is checked, this introductory-looking language sits at the exact place where the author is already stating the hukm and moving into the proof stream. The passage therefore behaves less like a neutral introductory preface and more like a compressed cluster:
- a chapter-specific introduction sentence
- a ruling statement
- a proof-overview claim
- a refutational move

Flattening that into one “introduction entry” would misclassify what the text is doing.

# Why the excerpt fails as one standalone entry

1. **It is too broad as one entry.**
2. **Its opening sentence misleads function detection.**
3. **The later clauses are not negligible enough to remain one pure-function unit.**
4. **Blind splitting would create self-containment failures because of linking expressions.**
5. **Keeping it fully merged would hide the distinction between introduction, hukm, proof-overview, and refutation.**

So the passage fails in two opposite ways at once:
- too merged if kept whole
- unsafe if split blindly

# Sub-excerpt map

## Candidate 1 — chapter-specific introduction
Text core:
- `بَاب المسْح عَلى الخفّين`
- `هذا الباب يذكر فيه شيء من أدلة مشروعية المسح على الخفين`

Function:
- chapter introduction
- not a full-topic introduction

Placement:
- introduction / chapter-intro layer
- not the general topic-introduction layer for المسح على الخفين as a whole

## Candidate 2 — ruling excerpt
Text core:
- the retained chapter-specific lead-in
- `لأن المسح عليهما بدل غسلهما`

Function:
- hukm entry

## Candidate 3 — proof overview

Function:
- proof-overview / legal-basis overview

## Candidate 4 — refutation
Text core:
- `ولا يعتبر شذوذ بعض الطوائف ...`

Function:
- refutational move

Complication:
- the closing phrase returns toward hukm/praise language, so refutation is not perfectly isolated

# Title-retention analysis

## Where title retention is required
For the introduction excerpt, the title is required because `هذا الباب` points back to it directly. Removing the title would damage self-containment.

## Where title retention is beneficial
For the ruling excerpt, the title is not strictly required for bare sentence-level intelligibility, but it preserves an important contextual fact: the ruling was stated right at the chapter start. That gives added understanding about where the author places the hukm.

## What would be reckless
Treating title retention as either always optional or always required. The case is asymmetric.

# Linking-word dependency analysis

This case exposes several dependency types:

- `هذا الباب` -> chapter-reference dependency
- `لأن` -> causal continuation dependency
- `عليهما` -> referential dependency
- `غسلهما` -> referential dependency
- `فهو` / `فيها` -> downstream semantic anchoring pressure

The protocol lesson is not “never split.” The lesson is: split only after dependency analysis.

# Forgiving-rule analysis

The owner’s logic here is hierarchical:

## Stage 1: first split pressure
The first sentence is its own functionally meaningful unit. It deserves its own home.

## Stage 2: forgiving retention
Even though the first sentence has its own function, it may still remain attached inside the hukm excerpt because:
- the carryover is small
- the carryover is strongly linked
- cutting it away would expose `لأن` and related references
- a separate explanatory add-on would likely be worse than preserving the tiny upstream context itself

## Stage 3: no over-forgiveness
Later, once the body is seen to contain roughly ruling / proof-overview / refutation at substantial size, forgiving retention no longer justifies keeping the whole passage merged. That is where real sub-excerpting pressure takes over.

# Source-scope mismatch analysis

The opening sentence is not simply “an introduction to المسح على الخفين” in the full-topic sense. It is an introduction to a chapter whose scope is narrower and proof-oriented:
- `شيء من أدلة مشروعية المسح على الخفين`

That creates an understanding risk:
- if this gets placed as a general topic introduction
- the reader may incorrectly infer that the subject itself is exhausted by proof-introduction language

So the passage carries a **source-scope mismatch risk**:
- chapter-specific intro
- mistaken as full-topic intro

# What should split

These should not remain one monolithic entry:
- introduction
- hukm
- proof-overview
- refutation

# What should remain glued

At least conditionally:
- title with chapter-intro
- chapter-intro sentence retained in the hukm excerpt
- small linked wording that prevents unsafe starts at `لأن`
- any short hukm-return phrase that would otherwise be lost from the hukm home

# What would be reckless to flatten

1. Calling the whole passage an introduction.
2. Treating the whole passage as one clean ruling entry.
3. Splitting every segment at its first functional distinction without preserving links.
4. Ignoring the difference between a chapter-specific intro and a full-topic intro.
5. Treating title retention as a mere cosmetic preference.

# What would be reckless to automate blindly

1. Function detection by surface chapter-start cues alone.
2. Splitting at punctuation without dependency analysis.
3. Dropping short context clauses because they look small.
4. Keeping a multi-function block merged because it is short.
5. Duplicating a whole merged passage across multiple leaves instead of splitting it where the later segments become substantively large.
