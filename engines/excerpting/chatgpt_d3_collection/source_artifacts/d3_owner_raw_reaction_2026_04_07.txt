Because, as I said earlier: we need to enforce as many security and verification gates as possible to ensure SURGICAL ACCURACY. Excerpting is the most important yet dangerous part of the pipeline, so absurd amount of security and verification is absolutely justified.

AN IMPORTANT TOPIC: we have already established the rule that none of the authors original text should be lost (except maybe in very rare cases we may come across, but for now the base rule stands). But there is another important question: where do we draw the line of a piece of text being ‘unsignificant’ / ‘redundant’?

For example: imagine a part of text strictly speaking, this belongs to “proofs” … but what added benefit would it give to go through copying this, giving it its own dedicated excerpt, placing it at a tree … Especially since the system should recognize that this is a mere ‘mention’ of something that will come in more detail: the ayah itself will definitely still get mentioned and explained … so this is not the only reference of this proof.

So the point is: if a small piece of ‘quick mention of a proof’ was the only reference to the proof, then yes, it should definitely get its own entry. But imagine if we know it’s not; we know the book later brings up the exact ayah and goes into way more detail. Then, that short mention loses significance …

وهذا التفسير للكلالة، وهو تفسير أبي بكر الصديق، وعليه جمهور الصحابة والتابعين والأئمة، في قديم الزمن وحديثه، والفقهاء السبعة، والأئمة الأربعة رضي الله عن الجميع.

As you can see, attributions is too closely related to the definition and the proof. The definition and the proof are directly related to the attributor: it is the proof and definition themselves that get attributed; THEY ARE THE ATTRIBUTIONS.

So either we can keep the full excerpt (as I did here), or we can only keep the attribution-text and fill the rest up with context .. though in this specific case where the proof and definition are not exhaustively long (short and harmless), we can just keep them as one excerpt.

But if the proofs was longer (which happens often; proofs are long), then the attributions-text should not include that proof, and instead, filled with context.

You see how context can be replaced with actual text incases where the text is short and harmless? Else we need to manually fill with context.

This brings up the topic of attributions, and what is relevant to know when something gets attributed to someone.
